Packet architecture

All the communication between the server and the client is done with packets.

A packet is composed of 3 parts :

  • long int id : The unique identifier of the packet
  • int sequence : The sequence of the packet
  • short replication : The replication of the packet
  • std::time_t timestamp : The timestamp of the packet
  • char message[256] : The message of the packet
  • PacketType type : The type of the packet
  • Status status : The status of the packet
  • RFC::data data : The data of the packet

Each type of packet has a different data structure.

Packet types

Server

CONNECT

The server sends a CONNECT packet to all the clients when a new client is connected.

It will receive CONNECT from a client to accept a new connection. If the server can't accept a new connection, the server will send back a CONNECT packet with the status set to KO only to the sender, with an appropriate message.
