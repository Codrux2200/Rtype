enum RFC::PacketType

Type of the packet It determines the content of the packet, like the type of the data and the size of the packet.

Enumerator
CONNECT 

CONNECT packet data is a HubData. The server sends a CONNECT packet to all the clients when a new client is connected.

It will receive CONNECT from a client to accept a new connection.

If the server can't accept a new connection, the server will send back a CONNECT packet with the status set to KO only to the sender, with an appropriate message.

DISCONNECT 

DISCONNECT packet The data is stored on @red RFC::Packet::disconnectData "disconnectData" wich is of type HubData. The server sends a DISCONNECT packet to all the clients when a client is disconnected.

It will receive DISCONNECT from a client to disconnect it.

The data contained in the data must be the currently connected players without the disconnected one.

START 

START packet The data is stored on startData wich is of type StartData. The server sends a START packet to all the clients when the host starts the game.

It will receive START from a client to start the game.

The data contained in the data must be the map to be used.

It have to be sent by the host to the server. Then, the server will send it back to all the players to tell them that the game is about to start.

DEAD 

Sent by server to all the clients when a player is dead.
